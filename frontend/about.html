About This Tool

This is a configurable FAIR assessment framework that evaluates research datasets against the FAIR principles — Findable, Accessible, Interoperable, and Reusable. Unlike existing tools that apply fixed metrics, this tool allows users and communities to define their own evaluation profiles tailored to their domain-specific requirements.

Users can configure which identifier types, metadata fields, file formats, vocabularies, licenses, and provenance fields are required for their domain. The tool then evaluates a dataset against these criteria and produces a detailed report with scores, evidence, and actionable recommendations.

Indicator Priority Levels

Each FAIR metric is assigned a priority level based on the FAIR Data Maturity Model (Bahim et al., 2020). These levels indicate the relative importance of each indicator in achieving FAIRness.

Priority Definition
Essential Addresses an aspect of utmost importance to achieve FAIRness under most circumstances. FAIRness would be practically impossible to achieve if this indicator were not satisfied.
Important Addresses an aspect that might not be of the utmost importance under specific circumstances, but its satisfaction would substantially increase FAIRness.
Useful Addresses an aspect that is nice-to-have but is not necessarily indispensable for achieving FAIRness.
Reference: Bahim, C., Casorrán-Amilburu, C., Dekkers, M., Herczog, E., Loozen, N., Repanas, K., Russell, K., & Stall, S. (2020). The FAIR Data Maturity Model: An Approach to Harmonise FAIR Assessments. Data Science Journal, 19, 41. https://doi.org/10.5334/dsj-2020-041

Metric Weights

When calculating scores, each metric is weighted according to its priority level. Essential metrics contribute more to the final score than important or useful ones, reflecting their relative importance in the FAIR Data Maturity Model.

Priority Weight Effect on score
Essential 1.0 Full contribution to principle score
Important 0.75 75% contribution to principle score
Useful 0.5 50% contribution to principle score

Each metric result also carries a status: Pass = full weight, Partial = 50% of weight, Fail = 0. The principle score (F, A, I, R) is the weighted average of all metrics under that principle. The overall score is the average of the four principle scores.

Note: Because profiles are configurable, scores reflect FAIRness relative to the selected profile's requirements — not a universal absolute measure. The same dataset may score differently under different profiles. This is by design, as different communities have different FAIR requirements.

Maturity Levels

Overall scores are interpreted as maturity levels to give users a qualitative sense of where their dataset stands and what to focus on next.

Incomplete
0 – 39%
Does not yet meet basic FAIR requirements
Initial
40 – 59%
Basic FAIR elements present but significant gaps remain
Intermediate
60 – 79%
Core requirements met, notable gaps to address
Advanced
80 – 100%
Strong FAIRness, focus on remaining gaps

FAIR Metrics Reference

The following metrics are evaluated. Fixed metrics are always evaluated regardless of profile configuration. Configurable metrics adapt based on the selected profile.

ID Principle Description Priority Type
F1 Findable Dataset has a globally unique persistent identifier Essential Configurable
F2 Findable Rich metadata is provided to allow discovery Essential Configurable
F3 Findable Metadata includes the identifier for the data Essential Fixed
F4 Findable Metadata is indexed in a searchable resource Essential Configurable
A1 Accessible Metadata is retrievable via HTTP/HTTPS Essential Fixed
A1.1 Accessible Metadata uses a free and open standard protocol Essential Fixed
A1.2 Accessible Protocol supports authentication if data is restricted Useful Fixed
A2 Accessible Metadata persists even if data is no longer available Essential Fixed
I1 Interoperable Metadata uses a formal knowledge representation language Important Configurable
I2 Interoperable Metadata uses FAIR-compliant vocabularies Important Configurable
I3 Interoperable Metadata includes qualified references to other resources Important Configurable
R1 Reusable Metadata has a plurality of accurate and relevant attributes Essential Configurable
R1.1 Reusable Metadata includes a license for reuse Essential Configurable
R1.2 Reusable Metadata includes provenance information Important Configurable
R1.3 Reusable Metadata meets domain-relevant community standards Essential Configurable

Important Notes on Score Interpretation

Scores should be interpreted as a process tool for identifying weaknesses and guiding improvement — not as a definitive ranking of dataset quality. As argued by Azevedo and Dumontier (2020), absolute FAIRness is not achievable and aggregated FAIR scores are not universally validated measures.

Because this tool is configurable, two datasets with the same score under different profiles are not directly comparable. Scores only have meaning in the context of the profile under which they were evaluated.

Some metrics may fail due to repository limitations rather than dataset quality issues. For example, vocabulary references (I2) or domain-specific metadata fields (F2) may not be exposed through a repository's API even if the dataset contains this information. Recommendations should be read with this in mind.
