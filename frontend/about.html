What This Tool Does

This tool automatically evaluates research datasets against the FAIR principles (Findable, Accessible, Interoperable, Reusable) and produces a scored, metric-level report with actionable recommendations. Unlike fixed-criteria tools such as F-UJI and FOOPS!, this tool supports configurable domain profiles allowing communities to declare which identifier types, metadata fields, file formats, controlled vocabularies, and licencing requirements apply to their domain. The same dataset assessed under an Agriculture profile and a Life Sciences profile will receive different, contextually meaningful scores reflecting each community's specific standards.

Metadata can be retrieved automatically from public repositories (DataCite, Zenodo, PANGAEA, Dataverse, schema.org endpoints) using a DOI, Handle, or ARK identifier, or supplied manually by uploading a metadata file in DataCite JSON, schema.org JSON-LD, DataCite XML, RDF/XML, or RDF Turtle format.

How Scoring Works

Each of the fifteen FAIR metrics produces a result of pass, partial, or fail. Results are weighted by priority level, following the FAIR Data Maturity Model [1]:

Priority Weight Pass earns Partial earns Fail earns
Essential 1.01.00.50.0
Important 0.750.750.3750.0
Useful 0.50.50.250.0

Per-principle scores (F, A, I, R) are calculated as the ratio of earned weight to total possible weight, expressed as a percentage. The overall score is the unweighted average of the four principle scores.

Maturity Levels

LevelScore rangeInterpretation
Incomplete 0 – 39% Core FAIR requirements are not met.
Initial 40 – 59% Basic FAIR requirements are partially satisfied.
Intermediate 60 – 79% Most FAIR requirements are met with some gaps remaining.
Advanced 80 – 100% Dataset meets the FAIR requirements of the selected profile.
